# Coding Convention

이 문서는 Calen 프로젝트의 코딩 컨벤션과 작업 전 확인 절차를 기록하는 기준 문서입니다.

## 문서 관리 규칙

- 사용자가 앞으로 "코딩 컨벤션을 추가해달라"고 요청하면 해당 내용은 반드시 이 파일에 누적 기록합니다.
- 기존 컨벤션은 사용자가 명시적으로 수정 또는 삭제를 요청하지 않는 한 임의로 지우지 않습니다.
- 새 컨벤션을 추가할 때는 가능한 한 날짜와 적용 범위를 함께 남깁니다.

## 코드 작성 전 필수 절차

코드를 작성하거나 수정하기 전에는 반드시 다음 순서를 따릅니다.

1. 이 `codingconvention.md` 파일을 먼저 확인합니다.
2. 요청과 관련된 기존 코드, 문서, 설정을 확인해 현재 구현 방식과 충돌하지 않는지 검증합니다.
3. 적용 가능한 컨벤션을 기준으로 구현 방향을 정리합니다.
4. 코드를 작성한 뒤 요청 범위에 맞는 검증을 수행합니다.
5. 수행한 작업과 검증 결과를 `worklog.md`에 기록합니다.

## 기본 작업 원칙

- 기존 프로젝트 구조와 코드 스타일을 우선합니다.
- 불필요한 대규모 리팩터링은 피하고, 요청 범위에 맞게 수정합니다.
- 프론트엔드, 백엔드, 인프라 변경은 각각 관련 파일과 실행 흐름을 확인한 뒤 수정합니다.
- 검증은 변경 범위에 맞는 빌드, 테스트, 정적 확인, 수동 확인 중 가능한 방법으로 수행합니다.
- 검증을 실행할 수 없었던 경우에는 그 이유와 대체 확인 내용을 `worklog.md`에 남깁니다.

## Git 작업 규칙

- 작업 또는 기능이 하나 완료되면 커밋합니다.
- 커밋은 완료된 기능 또는 작업 단위로 나누어 작성합니다.
- 커밋 전에는 변경 파일을 확인하고, 현재 작업과 무관한 변경은 커밋에 포함하지 않습니다.
- 푸시는 사용자가 명시적으로 지시한 경우에만 수행합니다.
