# UI 대비 점검 매트릭스

이 문서는 TravelLedger의 모든 주요 화면에서 글자와 배경 대비를 확인하기 위한 페이지별 매트릭스다.
목표는 기본 테마, 다크 테마, 모바일, 데스크톱 환경 모두에서 텍스트가 보이는지 확인하는 것이다.

## 공통 확인 기준

- 제목, 본문, 보조 설명, 라벨, placeholder, 버튼 텍스트가 보이는가
- hover, focus, selected, active, disabled, readonly 상태에서도 보이는가
- 모달, 드로어, 팝업, 바텀시트 안에서도 보이는가
- 모바일에서 줄바꿈 없이 잘리거나 카드 밖으로 나가지 않는가
- 이미지, 지도, 썸네일 위 텍스트가 배경과 구분되는가
- 표, 리스트, 카드형 모바일 테이블의 라벨과 값이 모두 보이는가

## 점검 결과 기록 규칙

- 각 칸은 `통과`, `수정 필요`, `확인 불가` 중 하나로 기록한다.
- `수정 필요`인 경우 화면명, 테마, viewport, 문제 텍스트, 배경 위치를 함께 기록한다.
- 스크린샷이 있으면 파일명 또는 링크를 함께 남긴다.
- 수정 후 같은 조건으로 다시 확인하고 `통과`로 변경한다.

## 최소 viewport 기준

- 모바일: 390px x 844px
- 태블릿: 768px x 1024px
- 데스크톱: 1440px x 900px

## 완료 기준

- 모든 주요 화면이 기본/다크 테마에서 `통과` 상태여야 한다.
- 모바일과 데스크톱 중 하나라도 `수정 필요`가 남으면 완료로 보지 않는다.
- 자동 빌드 통과만으로는 완료가 아니며, 실제 렌더링 확인이 필요하다.
