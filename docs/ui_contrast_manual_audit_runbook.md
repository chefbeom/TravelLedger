# UI 대비 수동 검수 Runbook

이 문서는 TravelLedger의 모든 주요 화면에서 글자와 배경 대비를 실제 브라우저로 확인하는 절차다.
목표는 기본 테마와 다크 테마, 모바일과 데스크톱 환경 모두에서 텍스트가 읽히는지 증거를 남기는 것이다.

## 준비

- 프론트엔드 개발 서버 또는 운영 배포 화면을 연다.
- 로그인 가능한 일반 사용자 계정과 관리자 계정을 준비한다.
- 브라우저 DevTools에서 다음 viewport를 준비한다.
- 모바일: 390 x 844
- 태블릿: 768 x 1024
- 데스크톱: 1440 x 900

## 공통 실행 순서

1. 기본 테마로 전환한다.
2. 데스크톱 viewport에서 대상 화면을 연다.
3. 제목, 본문, 라벨, placeholder, 버튼, 배지, 링크, 표, 카드, 상태 메시지를 확인한다.
4. hover, focus, active, selected, disabled, readonly 상태를 가능한 범위에서 확인한다.
5. 모바일 viewport로 변경하고 같은 화면을 다시 확인한다.
6. 다크 테마로 전환한 뒤 2~5단계를 반복한다.
7. 문제가 있으면 스크린샷과 함께 `docs/ui_contrast_audit_matrix.md`에 `수정 필요`로 기록한다.
8. 문제가 없으면 같은 파일에 `통과`로 기록한다.

## 화면별 검수 절차

### 인증

- 로그인 화면에서 ID, 비밀번호, PIN 입력, 오류 메시지를 확인한다.
- 회원가입 또는 초대 링크 화면에서 도움말, 검증 메시지, PIN 입력 상태를 확인한다.
- 관리자 추가 인증 모달에서 제목, 설명, 입력 라벨, placeholder, 확인 버튼을 확인한다.

### 메인

- 런처 카드의 메뉴명, 숫자, 설명, 배지를 확인한다.
- 상단 테마/모바일 전환 버튼의 활성/비활성 상태를 확인한다.
- 대시보드 카드의 제목, 보조 텍스트, 수치가 배경과 분리되는지 확인한다.

### 가계부

- 월 달력에서 요일, 날짜, 선택 날짜, muted 날짜, 수입/지출 표시를 확인한다.
- 거래 시트에서 거래 제목, 시간, 구분, 카테고리, 결제수단, 금액, 수정/삭제 버튼을 확인한다.
- 금액 입력 패널에서 금액 표시, 빠른 금액 버튼, 날짜/시간 입력, select 옵션을 확인한다.
- 모바일에서 거래 시트가 카드형으로 보일 때 라벨과 값이 모두 보이는지 확인한다.

### 통계 및 AI

- 통계 차트의 축 라벨, 범례, 툴팁, 수치 카드, 표를 확인한다.
- AI 분석 패널에서 provider 배지, 준비/요청/오류/성공 상태 문구를 확인한다.
- AI 결과 모달에서 섹션 제목, 본문, 목록, 카드, PDF 저장/인쇄 버튼을 확인한다.
- PDF 인쇄 미리보기에서 배경 제거 후에도 글자와 표가 보이는지 확인한다.

### 여행

- 여행 허브 카드의 제목, 수치, 버튼, 배지를 확인한다.
- 지도 화면에서 Leaflet 줌 버튼, 레이어 컨트롤, attribution, 팝업 텍스트를 확인한다.
- 사진 지도 핀과 군집 핀에서 숫자, 썸네일, 오버레이 텍스트가 보이는지 확인한다.
- 공유 여행 페이지에서 지도 드로어, 사진 타일, 스토리 카드, 빈 상태를 확인한다.

### 드라이브

- 파일 카드와 파일 목록에서 파일명, 확장자, 메타 정보, 잠금/공유 배지를 확인한다.
- 상세 패널에서 권한, 공유 링크, 버전 정보, readonly 값이 보이는지 확인한다.
- 컨텍스트 메뉴, 버전 드로어, 이동 대상 목록에서 hover/disabled 상태를 확인한다.

### 관리자

- 관리자 요약 대시보드에서 카드 제목, 숫자, 상태 배지, 연결 버튼을 확인한다.
- AI 및 서버 제어판에서 서버 선택, 서버 추가, 모델 입력, 상태 메시지, 저장 버튼을 확인한다.
- 문의 관리에서 목록, 상세, 답변 입력, 상태 변경 버튼을 확인한다.
- 접근 및 사용자 관리에서 차단 IP, 로그인 기록, 사용자 상태, 초대 링크 목록을 확인한다.

### 알림

- 알림 목록에서 읽음/안읽음 상태, 제목, 본문, 날짜, 상태 배지를 확인한다.
- 페이지네이션 버튼의 현재 페이지, disabled 상태를 확인한다.
- 빈 상태와 오류 상태의 안내 문구가 보이는지 확인한다.

## 접근성/브라우저 설정 확인

- 키보드 Tab 이동으로 focus-visible 링이 보이는지 확인한다.
- skip-link가 포커스될 때만 보이는지 확인한다.
- Windows 고대비 또는 브라우저 forced-colors 모드에서 주요 화면을 확인한다.
- prefers-contrast: more 환경에서 배경과 글자 대비가 유지되는지 확인한다.
- 투명도 감소 환경에서 모달/토스트/드로어 배경과 글자 대비가 유지되는지 확인한다.

## 증거 기록 형식

문제가 있으면 다음 형식으로 남긴다.

```text
화면: 관리자 추가 인증 모달
조건: 모바일 390x844 / 기본 테마
문제: 제목이 흰 배경에 너무 연하게 표시됨
위치: 모달 상단 h2
증거: screenshots/contrast/admin-pin-mobile-light.png
상태: 수정 필요
```

수정 후에는 같은 조건으로 다시 확인하고 `상태: 통과`를 남긴다.

## 완료 기준

- `docs/ui_contrast_audit_matrix.md`의 모든 필수 화면이 기본/다크/모바일/데스크톱에서 `통과`여야 한다.
- 빌드 성공만으로는 완료가 아니다.
- 스크린샷 또는 명시적인 검수 기록이 없는 항목은 `미확인`으로 유지한다.
