# UI 대비 및 글자 가독성 점검 체크리스트

이 문서는 TravelLedger 화면에서 배경과 글자색이 겹쳐 보이지 않는 문제가 재발하지 않도록 확인할 기준을 정리한다.
기본 테마와 다크 테마, 모바일과 데스크톱을 모두 같은 수준으로 점검한다.

## 필수 점검 범위

- 공통 레이아웃: 상단바, 런처, 대시보드, 카드, 패널, 버튼, 링크
- 인증 화면: 로그인, 회원가입, PIN 입력, 관리자 추가 인증 모달
- 가계부 화면: 월 달력, 거래 시트, 금액 입력, 수입/지출 배지, 집계 카드
- 통계/AI 화면: AI 분석 패널, 결과 모달, PDF/인쇄용 결과, 히스토리 목록
- 여행 화면: 지도, Leaflet 컨트롤, 사진 핀, 군집 핀, 사진/썸네일 오버레이, 여행 공유 페이지
- 드라이브 화면: 파일 카드, 파일 목록, 상세 패널, 공유/권한/버전 드로어
- 관리자 화면: 제어판, 문의, 사용자 상태, 차단 IP, 로그인 기록, 초대 링크
- 알림 화면: 알림 목록, 읽음/안읽음 상태, 페이지네이션
- 보조 상태: 로딩, 빈 상태, 오류/성공/경고, disabled, readonly, selected, hover, focus

## 테마별 기준

- 기본 테마에서는 밝은 배경 위 본문 텍스트가 충분히 진해야 한다.
- 다크 테마에서는 어두운 배경 위 본문 텍스트가 충분히 밝아야 한다.
- primary, active, danger 버튼은 내부 `span`, `svg`, `path`까지 배경과 대비되어야 한다.
- placeholder, hint, meta, small 텍스트는 흐리더라도 읽을 수 있어야 한다.
- 이미지/지도 위 텍스트는 반드시 반투명 배경, 그림자, 또는 배지를 가진다.

## 모바일 기준

- 작은 화면에서 텍스트가 카드나 버튼 밖으로 잘리면 안 된다.
- 버튼과 탭은 최소 44px 터치 높이를 유지한다.
- 테이블이 카드형으로 바뀌는 경우 `data-label` 라벨이 명확히 보여야 한다.
- 모달, 바텀시트, 드로어는 내부 스크롤과 줄바꿈이 가능해야 한다.
- 날짜/시간/파일 선택 같은 네이티브 입력 컨트롤도 글자가 보여야 한다.

## 접근성 상태 기준

- `:focus-visible`은 기본/다크 테마 모두에서 명확히 보여야 한다.
- `skip-link`는 평상시 숨겨지고, 포커스 시 명확히 보여야 한다.
- `sr-only`, `visually-hidden` 텍스트는 대비 보정 때문에 화면에 노출되면 안 된다.
- `prefers-contrast: more`, `forced-colors: active`, `prefers-reduced-transparency: reduce` 환경에서도 텍스트가 보여야 한다.
- 인쇄/PDF 저장 시 배경이 제거되어도 글자와 표가 보여야 한다.

## 회귀 방지 규칙

- 새 컴포넌트는 기본 테마와 다크 테마 색상을 함께 정의한다.
- `opacity`로 disabled 상태를 표현하더라도 텍스트는 읽을 수 있는 수준을 유지한다.
- `color: transparent`, `-webkit-text-fill-color: transparent`는 실제 숨김 목적이 아니면 사용하지 않는다.
- 이미지/지도/썸네일 위에는 일반 텍스트를 직접 올리지 않는다.
- 모달/드로어/팝업 안의 버튼은 본문 색상 보정에 덮이지 않도록 별도 컨트롤 색상을 둔다.
- 모바일 반응형 표는 라벨과 값이 모두 읽히는지 확인한다.

## 수동 확인 시나리오

1. 기본 테마 데스크톱에서 관리자 추가 인증 모달의 제목, 설명, 라벨, 입력 placeholder, 버튼을 확인한다.
2. 다크 테마 데스크톱에서 관리자 제어판, 문의 목록, 로그인 기록 표를 확인한다.
3. 기본 테마 모바일에서 가계부 월 달력, 거래 시트, 금액 입력 패널을 확인한다.
4. 다크 테마 모바일에서 여행 지도 핀, 사진 썸네일 오버레이, Leaflet 컨트롤을 확인한다.
5. AI 분석 결과 모달에서 본문, 섹션 제목, 표, 코드/JSON 형태 텍스트, PDF 인쇄 화면을 확인한다.
6. 드라이브 파일 목록에서 파일명, 메타 정보, 공유 상태, disabled/readonly 상태를 확인한다.
7. 알림 목록에서 읽음/안읽음 상태, 페이지네이션, 빈 상태, 오류 상태를 확인한다.
